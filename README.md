# TFG_CPP: Implementation and analysis of BCH codes over noisy channels

This project is a final degree project focused on the **from-scratch implementation of BCH codes** and their **performance analysis over noisy channels**. The goal is to study how the error rate changes when information is transmitted over a hard-decision AWGN channel, comparing the coded and uncoded cases.

## What It Includes

- Finite-field arithmetic over `GF(2^m)`.
- BCH encoding.
- BCH decoding.
- Transmission simulation over a noisy channel.
- Measurement of `BER`, `CWER`, and average encoding/decoding times.
- Export of results to CSV for later analysis.

## Structure

- `src/main.cpp`: main simulation program.
- `src/core/`: implementation of `BCH_Codec`, `GaloisField`, `Polynomial`, and `Channel`.
- `include/`: public headers.
- `results/csv/`: results generated by simulations.
- `scripts/`: external utilities for processing or plotting results.

## Requirements

- C++ compiler with C++17 support.
- `make`.
- Python 3.x, only if you plan to use the analysis or plotting scripts.

## Build

From the repository root:

```bash
make -j
```

This generates the executable in `build/bch_test`.

## Run

General form:

```bash
./build/bch_test <m> <t> [ebno_min] [ebno_max] [step]
```

Example:

```bash
./build/bch_test 4 2
```

You can also use the `Makefile` targets:

```bash
make run-m4
```

## Output

During execution, a table is shown with `Eb/N0`, `BER`, `CWER`, and the number of simulated codewords. A CSV is also generated in `results/csv/` with columns such as:

- `m`
- `t`
- `n`
- `k`
- `ebno_db`
- `ber`
- `cwer`
- `codewords`
- `avg_enc_us`
- `avg_dec_us`

## Important Notes

- The main focus of the project is transmission over **noisy channels**, not just the BCH code implementation.
- The simulation compares performance with and without coding.
- The project is designed to support experimental study of error curves, not only to validate the mathematical correctness of the code.
